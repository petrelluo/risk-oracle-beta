Part 1: Compound Interest

Start Simple: What Is Interest?

When you deposit money in a bank, the bank pays you interest — a reward for letting them use your money. The key question is: how often does the bank calculate and add that interest?

Each time interest is added, the new total becomes the base for the next round of interest. This snowball effect is called compounding.

Building the Formula: From Simple to Compound

Step 1 — Start with one year, compounded once (n = 1)

You invest $P at annual rate r. After 1 year you earn r·P in interest, so your total is:

A = P + r·P = P(1 + r)

After 2 years: A = P(1+r)(1+r) = P(1+r)²

After t years: A = P(1+r)t

Step 2 — Compound n times per year

Now suppose the bank splits the year into n equal periods (e.g., n = 12 for monthly). Each period:

  • The rate applied is r/n (the annual rate divided by the number of periods)
  • The money grows by the factor (1 + r/n)
  • In one full year this happens n times, so the factor is (1 + r/n)n
  • Over t years it happens n·t times total

A = P(1 + r/n)nt

Example: 6% monthly for 2 years → rate per period = 0.06/12 = 0.005, total periods = 12×2 = 24, so A = P(1.005)24

Step 3 — What if n grows without bound? (Continuous compounding)

Imagine compounding every second, every millisecond… infinitely often. Mathematically, we ask: what does P(1 + r/n)nt approach as n → ∞?

It turns out this limit has a beautiful answer. The key fact from calculus is:

limn→∞ (1 + 1/n)n = e ≈ 2.71828…

By a similar argument, (1 + r/n)n → er as n → ∞, so over t years:

(1 + r/n)nt → ert

So the final balance when compounding continuously is:

A = P · ert

This is the maximum possible amount you can earn — continuous compounding is the best-case scenario, and ert is why the number e appears in finance.

The Two Formulas Side by Side

A = P(1 + r/n)nt

Compounded n times per year

  • P = principal (starting amount)
  • r = annual rate as a decimal
  • n = times compounded per year
  • t = years
  • r/n = rate per period
  • nt = total number of periods
A = Pert

Compounded continuously

  • P = principal
  • e ≈ 2.71828
  • r = annual rate as a decimal
  • t = years
  • No n — happens infinitely often
  • Use ln to solve for t or r

What Does n Actually Change? A Comparison

Invest $1,000 at 6% for 1 year. Watch how more frequent compounding gives slightly more:

CompoundingnResult
Annually1$1,060.00
Semi-annually2$1,060.90
Quarterly4$1,061.36

Key insight:

Annual → Monthly is a noticeable jump.

Monthly → Continuous is almost nothing (+$0.16).

The formula A = Pert gives the theoretical ceiling — you literally cannot earn more than this.

Worked Examples

📘 Example 1 — Find the Balance (n times/year)

$5,000 at 4% annual, compounded quarterly, after 6 years.

Identify: P = 5000, r = 0.04, n = 4, t = 6
A = 5000(1 + 0.04/4)4·6 = 5000(1.01)24
A = 5000 × 1.26974 ≈ $6,348.67
📘 Example 2 — Find the Initial Investment

3% annual, compounded 6 times/year, balance after 8 years is $6,715.81. Find P.

6715.81 = P(1 + 0.03/6)6·8 = P(1.005)48
P = 6715.81 ÷ (1.005)48 ≈ 6715.81 ÷ 1.27049 ≈ $5,285.00
📘 Example 3 — Continuous, Find Balance

$7,004 at 4% compounded continuously for 15 years.

A = 7004 · e0.04 × 15 = 7004 · e0.6
e0.6 ≈ 1.82212  →  A = 7004 × 1.82212 ≈ $12,764.76
📘 Example 4 — Solve for Time

$8,385 at 3% continuous. How long to reach $11,318.57?

11318.57 = 8385 · e0.03t  →  11318.57/8385 = e0.03t
Take ln: ln(1.34986) = 0.03t
t = ln(1.34986) / 0.03 ≈ 0.2996 / 0.03 ≈ 10 years
⚠️ Common Mistakes:  r must be a decimal (6% → 0.06).  The exponent is nt, not just t.  For continuous compounding, use A = Pert — there is no n.

Part 2: Inequalities — When to Flip the Sign

The Core Rule

✅ Do NOT flip when:
  • Adding or subtracting anything
  • Multiplying/dividing by a positive number
  • Exponential base > 1 (e.g., 2, 3, e, 10)
  • Log base > 1
🔄 FLIP when:
  • Multiplying/dividing by a negative number
  • Exponential base is between 0 and 1 (e.g., ½, ⅓)
  • Log base is between 0 and 1

Why Does the Base Cause a Flip?

Key idea: When the base is > 1, the function is increasing — larger exponents give larger outputs, so the inequality stays the same direction. When the base is between 0 and 1, the function is decreasing — larger exponents give smaller outputs, so the inequality flips.

y = 2x goes UP ↗

23 = 8,   24 = 16   (bigger x → bigger y)

So if 2a < 2b, then a < b  (same direction)

y = (½)x goes DOWN ↘

(½)3 = ⅛,   (½)4 = 1/16   (bigger x → smaller y)

So if (½)a < (½)b, then a > b  (flipped!)

Exponential Inequalities

📗 Example 5 — Base > 1, No Flip

Solve: 3x < 20

Can't rewrite 20 as a power of 3, so take log₃ of both sides.
log₃(3x) < log₃(20)  →  x < log₃(20)
Change-of-base: log₃(20) = log(20)/log(3) ≈ 2.727
x < 2.727. Base 3 > 1 → no flip. Test: x = 0 → 1 < 20 ✓; x = 3 → 27 < 20? No ✓
📗 Example 6 — Base Between 0 and 1, Flip!

Solve: (⅓)n ≥ 15

Rewrite: (3−1)n ≥ 15  →  3−n ≥ 15
Take log₃:   −n ≥ log₃(15)
Divide by −1 → FLIP!   n ≤ −log₃(15) ≈ −2.465
n ≤ −2.465. Test: n = −3 → (⅓)−3 = 27 ≥ 15 ✓; n = 0 → 1 ≥ 15? No ✓
📗 Example 7 — Same-Base Shortcut

Solve: (½)n+2 < (½)5

Same base (½), so we can compare exponents directly — but base ½ < 1, so we must flip!
n + 2 > 5  →  n > 3

Logarithmic Inequalities

📕 Example 8 — Log Base > 1, No Flip

Solve: log₃(10x − 9) > 4

Exponentiate with base 3: 10x − 9 > 3⁴ = 81
10x > 90  →  x > 9. Base 3 > 1 → no flip.
📕 Example 9 — Log Base Between 0 and 1, Flip!

Solve: log½ x ≥ 5

Exponentiate with base ½: x ≥ (½)5 = 1/32
But base ½ ∈ (0,1) → decreasing → FLIP:   x ≤ 1/32
Domain: x > 0.   Answer: 0 < x ≤ 1/32
📕 Example 10 — Always Check for Extraneous Solutions!

Solve: log₆(6x) + log₆(x−1) = 2

Combine: log₆[6x(x−1)] = 2
Exponentiate: 6x(x−1) = 36  →  x² − x − 6 = 0  →  (x−3)(x+2) = 0
x = 3 or x = −2. Check: x = −2 makes log₆(−12) → undefined. Reject x = −2.
x = 3 only.
