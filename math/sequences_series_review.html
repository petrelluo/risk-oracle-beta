8.1 — Defining and Using Sequences & Series

A sequence is an ordered list of numbers. Each number is called a term. The nth term is written aₙ. A series is the sum of the terms of a sequence.

Key Vocabulary

TermMeaningExample
SequenceOrdered list of numbers3, 7, 11, 15, …
Term (aₙ)Each number in the lista₁=3, a₂=7, a₃=11
Finite sequenceHas a last term2, 4, 6, 8, 10
Infinite sequenceContinues without end2, 4, 6, 8, …
SeriesSum of sequence terms3 + 7 + 11 + 15
Partial sum SₙSum of first n termsS₃ = 3 + 7 + 11 = 21
Sigma (Summation) Notation

The Greek letter Σ (sigma) is used to write series compactly.

n Σ aᵢ = a₁ + a₂ + a₃ + … + aₙ i=1

Reading Sigma Notation: ∑ from i=3 to 7 of (2i + 1)

Start at i = 3: plug in 3, 4, 5, 6, 7 into the rule (2i + 1).

= (7) + (9) + (11) + (13) + (15) = 55

The index doesn't have to start at 1 — read the lower bound carefully.

The lower index tells you WHERE to start, not how many terms there are. ∑ from i=3 to 7 has 5 terms (3, 4, 5, 6, 7), not 7.
8.2 — Arithmetic Sequences & Series

An arithmetic sequence has a constant common difference d between consecutive terms.

Test: subtract any term from the next — if the result is always the same, it's arithmetic.

Explicit Formula

aₙ = a₁ + (n − 1)d

aₙ = nth term  ·  a₁ = first term  ·  d = common difference  ·  n = term number

Arithmetic Series (Partial Sum)

Sₙ = n/2 · (a₁ + aₙ) or Sₙ = n/2 · (2a₁ + (n−1)d)
Finding Common Difference & nth Term

Given a₁ = 5 and d = −3:   a₁ = 5, a₂ = 2, a₃ = −1, a₄ = −4, …

Find a₂₀:   a₂₀ = 5 + (20−1)(−3) = 5 − 57 = −52

When d is negative, terms decrease. The sequence is still arithmetic.
If you're given two terms but NOT a₁ — say a₄ = 10 and a₉ = 35 — you must FIRST find d from the gap, then back-solve for a₁. Don't assume the problem starts at term 1.

Strategy: Given aₘ and aₙ, find d and a₁

  1. Set up: aₙ − aₘ = (n − m) · d
  2. Solve for d
  3. Substitute back: a₁ = aₘ − (m − 1)d
  4. Now use the explicit formula for any term

Example: a₄ = 10, a₉ = 35 → 35 − 10 = (9 − 4)d → 25 = 5d → d = 5 → a₁ = 10 − 3(5) = −5

8.3 — Geometric Sequences & Series

A geometric sequence has a constant common ratio r between consecutive terms.

Test: divide any term by the previous — if the ratio is always the same, it's geometric.

Explicit Formula

aₙ = a₁ · rⁿ⁻¹

r = common ratio = aₙ/aₙ₋₁

Geometric Series (Partial Sum)

Sₙ = a₁ · (1 − rⁿ) / (1 − r) where r ≠ 1

If r = 1: Sₙ = n · a₁

r can be negative (alternating signs) or a fraction (decreasing terms). Both are still geometric.
If aₙ/aₙ₋₁ is not constant, it's NOT geometric — even if it looks like it might be. Always check at least 2 consecutive ratios.

Strategy: Given aₘ and aₙ, find r and a₁

  1. Use: aₙ/aₘ = r^(n−m)
  2. Solve for r by taking the (n−m)th root
  3. Then: a₁ = aₘ / r^(m−1)

Example: a₃ = 12, a₆ = 96 → 96/12 = r³ → r³ = 8 → r = 2 → a₁ = 12/2² = 3

Tricky Cases in Geometric Sequences

Negative Ratio (r < 0)

Sequence: 3, −6, 12, −24, …

r = −2. Terms alternate sign. Still geometric.

a₅ = 3 · (−2)⁴ = 3 · 16 = 48

Fractional Ratio (|r| < 1)

Sequence: 64, 32, 16, 8, …

r = 1/2. Terms decrease toward 0. Still geometric.

This type of sequence leads to infinite sums!

8.4 — Infinite Geometric Series

An infinite geometric series converges (has a finite sum) only when |r| < 1.

When |r| ≥ 1, the series diverges — there is no finite sum.

Sum of an Infinite Geometric Series

S = a₁ / (1 − r) , only when |r| < 1

This formula does NOT apply if |r| ≥ 1.

Worked Example

Find the sum: 12 + 8 + 16/3 + 32/9 + …

r = 8/12 = 2/3. Since |2/3| < 1, the series converges.

S = 12 / (1 − 2/3) = 12 / (1/3) = 12 · 3 = 36

Students often apply S = a₁/(1−r) without checking |r| first. If r = −2, for example, the formula gives a wrong answer. ALWAYS check |r| < 1 first.

Repeating Decimals as Infinite Series

Write 0.̄3̄6̄ (= 0.363636…) as a fraction:

= 0.36 + 0.0036 + 0.000036 + … → a₁ = 0.36, r = 0.01

S = 0.36/(1 − 0.01) = 0.36/0.99 = 36/99 = 4/11

8.5 — Recursive Rules for Sequences

A recursive rule defines each term using previous term(s). It requires an initial condition (value of a₁ or a₁ and a₂).

Arithmetic Recursive Rule

a₁ = [value] aₙ = aₙ₋₁ + d

Example: a₁ = 3, aₙ = aₙ₋₁ + 4

Gives: 3, 7, 11, 15, …

Geometric Recursive Rule

a₁ = [value] aₙ = r · aₙ₋₁

Example: a₁ = 5, aₙ = 3 · aₙ₋₁

Gives: 5, 15, 45, 135, …

Fibonacci-Like (Neither Arithmetic Nor Geometric)

a₁ = 1, a₂ = 1, aₙ = aₙ₋₁ + aₙ₋₂

Gives: 1, 1, 2, 3, 5, 8, 13, 21, … — differences are NOT constant; ratios are NOT constant. It is neither arithmetic nor geometric.

A recursive rule without an initial condition is incomplete — it could define many different sequences.
When converting a recursive rule to an explicit formula: if aₙ = aₙ₋₁ + d, it's arithmetic (use aₙ = a₁ + (n−1)d). If aₙ = r·aₙ₋₁, it's geometric (use aₙ = a₁·rⁿ⁻¹).

Critical Mistake Checklist

Check These Before Every Problem

  • Does the sequence actually start at a₁ = term 1, or is the first given term some aₘ?
  • Have you verified the sequence is arithmetic/geometric — or is it neither?
  • For geometric series: is |r| < 1 before applying the infinite sum formula?
  • For sigma notation: what is the lower index? It may not be 1.
  • For partial sums: have you counted the number of terms correctly?
  • Is r negative? Then even-powered terms are positive, odd-powered are negative.
  • Recursive rules: are you using the previous term, not the index?
